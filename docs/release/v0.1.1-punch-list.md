## Manual e2e (owner runs against tagged build)

1. `git clone --depth 1 --branch v0.1.0 git@github.com:kennygeiler/DeployAI.git deployai-test && cd deployai-test`
2. `make dev` — verify all 7 containers report healthy (control-plane, web, postgres, redis, minio, keycloak, freetsa-stub)
3. `make seed-app` — confirm seed tenant + engagement appear
4. Browser: visit `http://localhost:3000`
   - [ ] redirects to `/queues` or `/onboarding`
   - [ ] `/settings` loads + all 5 forms render
   - [ ] `/settings/audit` shows seed events
   - [ ] `/settings/email-import` paste form accepts a sample mbox
   - [ ] engagement-detail page renders matrix graph + insights
   - [ ] `/healthz` → 200, `/readyz` → 200, `/metrics` → prometheus text format
5. Backup cycle:
   - [ ] `S3_BUCKET=test S3_ENDPOINT_URL=http://localhost:9000 AWS_ACCESS_KEY_ID=minioadmin AWS_SECRET_ACCESS_KEY=minioadmin make backup` succeeds
   - [ ] `DEPLOYAI_RESTORE_CONFIRM=YES DEPLOYAI_RESTORE_FORCE_OVERWRITE=YES BACKUP=s3://test/deployai/backups/<ts> make restore` round-trips clean
   - [ ] `DEPLOYAI_PRUNE_CONFIRM=YES BACKUP_RETENTION_DAYS=0 make backup-prune` deletes the round-tripped folder
6. Export packet:
   - [ ] `make export ENGAGEMENT=<seed-engagement-uuid>` writes `packet/<id>/{markdown.md,packet.pdf,data.json}`
7. JSON log mode:
   - [ ] `LOG_FORMAT=json docker compose restart control-plane && curl localhost:8000/healthz && docker compose logs --tail 5 control-plane | jq .` → valid JSON per line
8. Request correlation:
   - [ ] `curl -H "X-Request-ID: $(uuidgen)" -i localhost:8000/healthz` echoes the id in response header + appears in JSON logs
